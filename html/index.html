What to say about this guy...

I always had an interest in most computer related things growing up. I've always been interested in knowing how everything worked from why the parts in the computer made noise and how the applications I use so often were made. Because of that I began studying computer science, learning the concepts of programming however I still had an interest in the hardware aspect of computer technology.

At the time I couldn't decide which field I had most interest in and I had to make a decision. I decided on tackling them both. Along the way I learned I have a huge interest in learning new things. I gained interests in game design, game development, web development, hardware interfacing (currently with a Raspberry Pi and Arduino board) and robotics. With each of these interests I find myself taking time to learn more about them when I can.

Currently I am interesting in finding a position where I can apply and learn more about the software development process. I've been learning about front-end web development and begun investigating on the back-end using Nodejs, as well as rebuilding a raspberry pi controlled robot written mostly in Python. I hope to find a position that would teach me more about software development or hardware interfacing.

Raspberry Pi Robot

I chose to rebuild a robot that me and my friends built together of the course of this past semster as a junior design project. We called it the RES Bot, Robots for Emergency Situations. The Purpose of RES 1.0 was to take some readings of its environment and report it back to the user via a web gui accessible from any browser. This bot would detect distance, temperature, light, and motion.

RES 2.0 will include most of that. The goal is to have automatic movement in addition to manual control which is what it had in the last version. Local area mapping is also a goal this time around. Possibly camera imaging but that one is a maybe.
